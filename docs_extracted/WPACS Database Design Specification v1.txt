WPACS Database Design Specification v1.0 
Workforce Productivity & Activity Correlation System 
Enterprise Data Architecture Specification 
Purpose 
This document defines the database architecture, entity relationships, table structures, indexing 
strategy, partitioning model, audit framework, and data retention requirements for WPACS. 
Target Database 
PostgreSQL 16+ 
 
1. DATABASE ARCHITECTURE 
PRINCIPLES 
Principle 1 
Event data is immutable. 
Events are never modified. 
Events may only be: 
 Inserted  
 Archived  
 
Principle 2 
Derived data is separated from source data. 
Source Events 
↓ 
Validation 

↓ 
Metrics 
↓ 
Reports 
 
Principle 3 
Every calculation is traceable. 
Every score must trace back to: 
 Rule  
 Event  
 Evidence

4. ORGANIZATIONS TABLE 
organizations 
CREATE TABLE organizations ( 
    organization_id UUID PRIMARY KEY, 
 
    name VARCHAR(255) NOT NULL, 
 
    code VARCHAR(50) UNIQUE, 
 
    status VARCHAR(50), 
 
    timezone VARCHAR(100), 
 
    created_at TIMESTAMP, 
 
    updated_at TIMESTAMP 
); 
 
5. DEPARTMENTS TABLE 
departments 

CREATE TABLE departments ( 
    department_id UUID PRIMARY KEY, 
 
    organization_id UUID NOT NULL, 
 
    name VARCHAR(255), 
 
    description TEXT, 
 
    created_at TIMESTAMP, 
 
    updated_at TIMESTAMP 
); 
 
Foreign Key 
organization_id 
REFERENCES organizations

26. INDEXING STRATEGY 
Mandatory Indexes 
CREATE INDEX idx_events_employee 
ON raw_events(employee_id); 
 
CREATE INDEX idx_events_timestamp 
ON raw_events(event_timestamp); 
 
CREATE INDEX idx_events_org 
ON raw_events(organization_id); 
 
CREATE INDEX idx_conflict_employee 
ON conflicts(employee_id); 
 
CREATE INDEX idx_productivity_employee 
ON productivity_scores(employee_id); 
 
27. JSONB INDEXING 

CREATE INDEX idx_events_payload 
ON raw_events 
USING GIN(payload);
