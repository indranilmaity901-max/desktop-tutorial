WPACS V2 Desktop Agent Implementation Notes

Status: V2 Windows Desktop Agent implemented as a lightweight Python background process in agent/windows_agent.py.

Purpose
The WPACS Desktop Agent submits Windows workstation state events to the WPACS API. It only reports approved operational state transitions needed for attendance, agent status, and productivity calculations.

Supported events
SHIFT_START
SHIFT_END
LOCK
UNLOCK
LOGIN
LOGOFF
HEARTBEAT

Event contract
Each agent event maps to the workstation_events table through the POST /api/v2/events contract:
employee_id
event_type
event_timestamp
source

Security boundaries
The desktop agent authenticates against the WPACS API before event submission.
The backend remains the system of record for RBAC, audit logging, productivity calculation, and manager visibility.
Managers may only view employees assigned to them.
Supervisors remain read-only and limited to assigned operational scope.

Explicit exclusions
The desktop agent must not capture screenshots.
The desktop agent must not log keystrokes.
The desktop agent must not track mouse movement.
The desktop agent must not collect browser history, application titles, document names, message content, screen contents, camera data, microphone data, or file contents.
The desktop agent must not implement surveillance features.

V1.5 dependency
The desktop agent uses the event foundation:
agent_status
workstation_events
productivity_daily
POST /api/v2/events
GET /api/v2/agent-status
GET /api/v2/productivity
WebSocket live updates at /api/v2/live
